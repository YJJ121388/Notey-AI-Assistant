# 📊 短视频知识管理用户行为与需求挖掘报告

| 报告属性 | 内容 |
| :--- | :--- |
| **报告性质** | **产品需求挖掘** |
| **调研日期** | 2025年12月20日 |
| **调研目标** | 探索短视频用户在“娱乐流”中进行“知识获取”的行为模式与痛点，为产品形态定义提供数据支撑。 |
| **核心发现** | **现有收藏机制失效，用户需要一个“零摩擦捕获”+“深度解析”+“本地化资产”的独立工具。** |

---

## 1. 摘要 

通过对 328 名泛知识学习人群的调研，确认了一个巨大的**市场空白**：用户在娱乐平台（抖音/小红书）上的“知识获取”需求旺盛，但缺乏有效的管理工具。

**关键问题**
1.  **机会**：**81%** 的用户渴望一种“不打断刷视频”的保存方式，这直接指向了 **“后台静默捕获”** 的产品形态。
2.  **方向**：**86%** 的用户认为评论区（课代表/避雷）价值极高却无法保存，这为我们后续开发 **“深度解析引擎”** 提供了核心依据。
3.  **架构**：用户对数据主权表现出强烈的掌控欲，**64%** 要求本地存储与编辑，这决定了我们必须采用 **“本地优先 ”** 的技术架构。

---

## 2. 需求分析 

### 2.1 痛点一：心流冲突与“零摩擦”需求
* **现象**：92% 的用户在娱乐时会刷到高价值干货，但 **67%** 的人因为“不想打断当前的娱乐节奏”而放弃深度记录，仅做无效收藏。
* **用户原声**：“有时候刷到好东西想记，但一想到要截图、裁剪、或者切出去打开笔记App，就懒得弄了，直接划走。”
* **👉 产品设计指引 (To V1.0)**：
    * 产品必须极致轻量，不能抢占前台。
    * **交互建议**：利用 iOS **Share Extension** 实现“点击即走”的后台处理，确保用户心流不中断。

### 2.2 痛点二：收藏夹失效与“结构化”需求
* **现象**：74% 的用户表示收藏夹是“黑洞”。核心原因是“内容杂乱(45%)”和“非结构化(38%)”。视频太长，用户无法快速检索到知识点。15%的用户曾遇到过收藏夹中内容因为权限变动等原因变为不可查看状态。
* **数据**：用户最想保存的内容 Top 3 是 **知识技能学习(78%)、消费推荐与避雷(65%)、旅游攻略(55%)、实时热点信息（45%）**。这些内容都具有极强的“结构化”特征（步骤、代码、清单）。
* **👉 产品设计指引 (To V1.0 & V2.0)**：
    * 仅仅保存链接是不够的，必须将视频内容 **“转译”** 为文字笔记。
    * **功能建议**：引入 LLM (大模型) 进行总结，针对菜谱、代码等特定类型输出 **Markdown 结构化笔记**。

### 2.3 痛点三：隐性信息丢失与“深度解析”需求
* **现象**：调研发现一个极具价值的盲点——**85%** 的用户认为“评论区”的信息（如避雷、补充、神回复）比视频本身更重要，但目前的收藏工具都无法保存评论。
* **👉 产品设计指引 (To V3.0)**：
    * 这是我们与竞品（如简单的链接转文字工具）拉开差距的关键。
    * **功能建议**：开发针对特定平台（抖音/小红书）的 **解析器**，专门抓取 **Top 3 高赞评论** 并合并到笔记中。

### 2.4 痛点四：信息孤岛与“智能拓展”需求
* **现象**：72% 的用户在保存“旅游攻略”或“消费评测”后，还需要手动去其他平台搜索“避雷指南”或“真实评价”，这一过程繁琐且割裂。
* **👉 产品设计指引 (To V3.0)**：
    * 笔记工具不能止步于“记录”，应该辅助“完善”。
    * **功能建议**：设计 **“智能追加”** 功能，利用 AI 自动联网搜索相关的高热补充信息，解决信息不全的问题。

### 2.5 痛点五：掌控感缺失与“本地化”需求
* **现象**：**91%** 的用户表示 AI 生成的内容一定需要“二次修改”；**54%** 的用户对纯云端笔记表示不信任，强烈要求“本地保存”和“导出能力”。
* **👉 产品设计指引 (To V2.0)**：
    * 产品不能做成封闭的“阅后即焚”工具，而应是“资产管理”工具。
    * **架构建议**：V2.0 阶段需引入 **本地数据库**，支持离线查看、编辑 Markdown，并提供明确的 **归档/分类** 管理能力。

## 3. 产品功能规划

基于上述调研数据，建议将产品迭代路径规划如下：

### 第一阶段：MVP 验证期 (对应 V1.0)
* **目标**：解决最核心的“心流冲突”问题，验证“极速捕获”的市场需求。
* **核心功能**：
    * 基于 Shortcuts 的 Share Extension 入口（零干扰）。
    * 基础的“视频转文字”总结能力。
    * **预期验证**：用户是否愿意用这种方式替代“截图/收藏”。

### 第二阶段：资产构建期 (对应 V2.0)
* **目标**：解决“收藏夹失效”和“掌控感缺失”问题，打造独立笔记容器。
* **核心功能**：
    * 独立 App 客户端。
    * **本地数据库 + Markdown 编辑器**（响应 91% 的编辑需求）。
    * 分类归档与状态管理（响应“收藏夹杂乱”痛点）。

### 第三阶段：智能增值期 (对应 V3.0)
* **目标**：解决“隐性信息丢失”和“信息孤岛”问题，构建核心壁垒。
* **核心功能**：
    * **评论区深度解析**（响应 85% 对评论价值的认可）。
    * **AI 联网智能追加**（响应 72% 对补充信息的需求）。

---

## 4. 结论 (Conclusion)

本次调研清晰地描绘了一个 **“不仅要快，更要深，且属于我”** 的用户需求画像。

现有的通用型笔记软件（如备忘录）太重，无法承接“刷视频”时的碎片化记录；而简单的链接保存工具又太轻，无法解决“内容结构化”的痛点。

**综上，我认为 Notey 的机会点在于：做一个“懂中国互联网内容”的本地化AI信息捕手。**
